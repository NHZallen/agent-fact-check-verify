# Agent Fact Check Verify

**مبدّل اللغة**: [中文](../README.md) | [English](README.en.md) | [Español](README.es.md) | **العربية (الحالي)**

الإصدار: **1.0.1**  
المؤلف: **Allen Niu**  
الترخيص: **MIT**

`agent-fact-check-verify` هي مهارة تحقق صارمة لوكلاء الذكاء الاصطناعي. تقوم باستخراج الادعاءات القابلة للتحقق، وتنفيذ تحقق متعدد المصادر (رسمي، إعلام رئيسي، مواقع تحقق، إشارات اجتماعية)، وتطبيق قواعد داخلية حتمية، ثم إنتاج رد محايد ومتكامل دون إظهار تفاصيل التقييم الداخلي للمستخدم.

---

## 1. أهداف التصميم والمبادئ المهنية

هذه المهارة مصممة لسير عمل قابل للتدقيق وإعادة التنفيذ، وليس لإنتاج نص يبدو مقنعًا فقط.

- **قابلة لإعادة الإنتاج**: نفس الأدلة تعطي نفس القرار.
- **قابلة للتتبع**: كل استنتاج مرتبط بمصدر قابل للنقر.
- **قابلة للتدقيق**: قواعد داخلية ثابتة دون تقييم عشوائي.
- **صياغة محايدة**: بدون لغة منحازة سياسيًا.
- **تكلفة محدودة**: ميزانية بحث محددة لكل ادعاء.

## 2. النطاق (ما الذي تفعله / لا تفعله)

### عبارات التفعيل (موصى بها)

إذا احتوى إدخال المستخدم على “verify” أو “fact-check” أو “is this true” أو “核實” أو “核實這個”، فيجب تفعيل هذه المهارة أولًا.

### 2.1 ما الذي تفعله

1. استخراج الادعاءات من النصوص الطويلة.
2. تصنيف الادعاء: statistical / causal / attribution / event / prediction / opinion / satire.
3. تحقق ثلاثي المراحل: رسمي → إعلام رئيسي → أدلة معاكسة.
4. قرار داخلي حتمي بنطاقات.
5. مخرجات مستخدم متكاملة بدون عرض التقييم.

### 2.2 ما الذي لا تفعله

1. لا تفرض حكم صحيح/خاطئ على الآراء الذاتية البحتة.
2. لا تعتبر كثافة النقاش الاجتماعي دليلًا رئيسيًا.
3. لا تنتج لغة إقناعية سياسية.
4. لا تضمن تغطية المحتوى الخاص أو المدفوع.

## 5. أدوات CLI الاختيارية وفئات ملفات الارتباط (مهم)

هذه الأدوات **اختيارية**. التدفق الأساسي يعمل بدونها.

- CLI لمنصة X: <https://github.com/jackwener/twitter-cli>
- CLI لمنصة Reddit: <https://github.com/jackwener/rdt-cli>

### 5.1 twitter-cli (وضع Cookie)

فئات ملفات الارتباط الشائعة:

- **مصادقة مطلوبة**: `auth_token`, `ct0`
- **مساندة الجلسة**: `guest_id`, `kdt`
- **حقول اختيارية**: `twid`, `lang`

ممارسات تشغيلية:

- حفظ ملفات الارتباط بصلاحيات مقيدة.
- عدم رفع ملفات الارتباط إلى git.
- تدوير ملفات الارتباط دوريًا.

### 5.2 rdt-cli (وضع Cookie)

فئات ملفات الارتباط/الجلسة الشائعة:

- **جلسة أساسية**: `reddit_session`
- **جهاز/تتبع**: `loid`, `session_tracker`
- **حقول مصادقة اختيارية**: `token_v2` (حسب الإصدار)

ممارسات تشغيلية:

- استخدام حساب منخفض الصلاحيات.
- تحديث ملفات الارتباط المنتهية وتجنب التخزين النصي المكشوف في البيئات المشتركة.

## 6. سير التنفيذ الموصى به

### الخطوة A: استخراج الادعاءات

```bash
python3 scripts/factcheck_engine.py extract \
  --text "نص الإدخال" \
  --output claims.json
```

### الخطوة B: تحقق ثلاثي المراحل (جهة الوكيل)

1. **ابدأ بالمصادر الرسمية/الأولية**.
2. **تحقق تقاطعي مستقل من الإعلام الرئيسي**.
3. **ابحث عن الأدلة المعاكسة/التفنيد**.

الحد الموصى به: 6 عمليات بحث لكل ادعاء. ويمكن أيضًا تنفيذ عمليات تحقق متعددة على X(Twitter) (الموصى به: 3 جولات) كتعزيز اجتماعي، من دون تغيير عدد عمليات البحث الرسمية/الإعلامية/المضادة.

### الخطوة C: القرار الداخلي

```bash
python3 scripts/factcheck_engine.py score \
  --input evidence.json \
  --output scored.json
```

### الخطوة D: توليد رد المستخدم

```bash
python3 scripts/factcheck_engine.py compose \
  --input scored.json \
  --output reply.txt
```

## 8. قواعد صارمة لمخرجات المستخدم

- لا تعرض الدرجة الداخلية.
- لا تكشف منطق التقييم الداخلي.
- لا تقسّم الرد إلى «معلومة إضافية» و«معلومة صحيحة»؛ ادمجها في سرد واحد.
- تجنب القوائم إلا عند الضرورة.
- استخدم روابط قابلة للنقر للمصادر.
- ترتيب العرض: خاطئ → غير مؤكد → صحيح.
- أضف دائمًا:

`⚠️ يعتمد هذا التحقق على المعلومات المتاحة علنًا ولا يمكنه تغطية المواد الخاصة أو المحجوبة خلف جدار دفع.`

---

## 9. التعامل مع الحالات الحدّية

- **Prediction**: بدون حكم صحيح/خاطئ؛ تلخيص توقعات المصادر المتاحة.
- **Opinion**: تصنيفها كطرح ذاتي خارج نطاق التحقق الحقائقي.
- **Satire**: تصنيفها كمصدر ساخر/خيالي.
- **أدلة غير كافية**: إرجاع «غير قابل للتحقق حاليًا» بشكل محافظ.

---

## 10. المخاطر والحدود

1. المعلومات العامة ليست كاملة دائمًا.
2. الأخبار العاجلة قد تتغير بسرعة.
3. الإشارات الاجتماعية مساعدة وليست دليلًا رئيسيًا.
4. حتى المصادر الرسمية قد تحمل انحيازًا مؤسسيًا، لذا يلزم التحقق المتقاطع.
