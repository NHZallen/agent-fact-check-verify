## 1. الهدف ومبادئ التصميم

تركز هذه المهارة على التحقق من الحقيقة وقابلية تتبع الأدلة. لا تنتج صياغة عاطفية أو إقناعًا سياسيًا.

مبادئ التصميم:

1. **قابل لإعادة الإنتاج**: نفس الأدلة تعطي نفس القرار.
2. **قابل للتتبع**: كل استنتاج مرتبط بمصادر.
3. **محايد**: الصياغة الموجهة للمستخدم تبقى محايدة.
4. **بسيط خارجيًا**: لا يتم كشف التقييم الداخلي.
5. **تكلفة محدودة**: ميزانية بحث ثابتة لكل ادعاء (الحد الموصى به: 6).

## 2. النطاق

- استخراج الادعاءات من النصوص الطويلة.
- تصنيف الادعاءات: statistical / causal / attribution / event / prediction / opinion / satire.
- تحقق متعدد المصادر: رسمي، إعلام رئيسي، أدلة معاكسة، إشارات اجتماعية.
- تقييم داخلي: نظام حتمي من 100 نقطة (داخلي فقط).
- رد موجّه للمستخدم: استنتاج مدمج بدون إظهار الدرجة.

## 3. المتطلبات غير الوظيفية

- لغة المستخدم الافتراضية: الصينية.
- ترتيب الرد: الخاطئ أولًا ثم غير المؤكد ثم الصحيح.
- تجنب النقاط إلا عند الحاجة.
- إضافة هذا التنبيه دائمًا:

`⚠️ يعتمد هذا التحقق على المعلومات المتاحة علنًا ولا يمكنه تغطية المواد الخاصة أو المحجوبة خلف جدار دفع.`

## 5. التثبيت

### 5.1 متطلبات النظام

- Python 3.10+
- سلسلة أدوات للوكيل مع إمكانية البحث على الويب (Brave / Tavily / Browser)

### 5.2 الصلاحيات

- صلاحية قراءة/كتابة في مساحة العمل
- القدرة على استدعاء أدوات البحث

### 5.3 تحقق أساسي

```bash
python3 scripts/factcheck_engine.py --help
```

## 6. أدوات CLI الاختيارية وفئات ملفات تعريف الارتباط

هاتان الأداتان **اختياريتان** وليستا إلزاميتين. عند عدم توفرهما يتم الرجوع إلى بحث الويب العادي.

- البحث في X: <https://github.com/jackwener/twitter-cli>
- البحث في Reddit: <https://github.com/jackwener/rdt-cli>

### 6.1 فئات ملفات تعريف الارتباط الشائعة لـ twitter-cli

الحقول الفعلية تعتمد على إصدار CLI. الفئات الشائعة:

- مطلوبة: `auth_token`, `ct0`
- مساعدة شائعة: `guest_id`, `kdt`
- قد تظهر: `twid`, `lang`

توصيات:

- خزّن ملفات الارتباط في بيئة محلية آمنة فقط.
- لا تقم أبدًا بإيداع ملفات الارتباط في نظام التحكم بالإصدارات.

### 6.2 فئات ملفات تعريف الارتباط/الجلسة الشائعة لـ rdt-cli

الحقول الفعلية تعتمد على إصدار CLI. الفئات الشائعة:

- الجلسة: `reddit_session`
- الجهاز/التتبع: `loid`, `session_tracker`
- حقول أخرى محتملة: `token_v2` أو ملف ارتباط مصادقة مكافئ

توصيات:

- فضّل تدفق OAuth الرسمي إذا كان مدعومًا.
- إذا كان تسجيل الدخول عبر ملفات الارتباط مطلوبًا، استخدم حسابًا بأقل صلاحيات وبدّل دوريًا.

## 7. سير العمل

### 7.1 استخراج الادعاءات

```bash
python3 scripts/factcheck_engine.py extract \
  --text "نص الإدخال" \
  --output claims.json
```

### 7.2 التحقق الخارجي (ينفذه الوكيل)

ثلاث جولات موصى بها:

1. مصادر رسمية وأولية
2. تقاطع مع الإعلام الرئيسي
3. أدلة معاكسة وعمليات التفنيد

ثم تجميع الأدلة في JSON وتشغيل التقييم.

### 7.3 التقييم الداخلي

```bash
python3 scripts/factcheck_engine.py score \
  --input evidence.json \
  --output scored.json
```

### 7.4 تركيب رد المستخدم

```bash
python3 scripts/factcheck_engine.py compose \
  --input scored.json \
  --output reply.txt
```

## 8. صيغ الإدخال والإخراج

### 8.1 إدخال score (evidence.json)

يمكن أن يتضمن كل ادعاء:

- `claim`
- `type`
- `evidence.official_count`
- `evidence.mainstream_count`
- `evidence.independent_count`
- `evidence.factcheck_true`
- `evidence.factcheck_false`
- `evidence.authority_rebuttal`
- `evidence.outdated_presented_current`
- `evidence.source_chain_hops`
- `evidence.core_contradiction`
- `evidence.has_timestamp`
- `evidence.strong_social_debunk`
- `evidence.out_of_context`
- `evidence.headline_mismatch`
- `evidence.missing_data_citation`
- `evidence.fact_opinion_mixed`

### 8.2 إدخال compose (scored.json)

- `band` من score: `true|false|uncertain|prediction|opinion|satire`
- يمكن للوكيل إثراء `findings`, `correct_info`, `sources`

## 9. سياسة القرار (بدون إظهار الدرجة للمستخدم)

- `true`: اذكر أنه صحيح مع سياق دقيق إضافي.
- `false`: اذكر عدم التطابق وادمج المعلومة الصحيحة مباشرة.
- `uncertain`: اذكر أن التحقق غير كافٍ حاليًا.
- `prediction`: بدون حكم صحة؛ اعرض مصادر التوقع المتاحة.
- `opinion`: اعتبره رأيًا خارج نطاق التحقق الحقائقي.
- `satire`: اعتبره مصدرًا ساخرًا/خياليًا.

## 10. قواعد أسلوب الرد

- لا تعرض الدرجة الداخلية.
- لا تفصل الرد إلى كتل "معلومة إضافية" أو "المعلومة الصحيحة"؛ ادمجها طبيعيًا.
- تجنب النقاط إلا عند الحاجة.
- اعرض النتائج الخاطئة أولًا ثم غير المؤكدة ثم الصحيحة.
- استخدم روابط قابلة للنقر للمصادر.

## 11. القيود والمخاطر

- المواد الخاصة/المدفوعة غير مرئية.
- الأخبار العاجلة قد تتغير خلال دقائق.
- الإشارات الاجتماعية ليست دليلًا أساسيًا.
- بعض البيانات الرسمية قد تحمل انحيازًا مؤسسيًا وتتطلب تحققًا متقاطعًا.
